3  Exploring Data with AI

Module 1 described how agents replace dashboards by letting you query data in plain English. This module puts that into practice. You will go from a raw spreadsheet to a written recommendation with charts in under ten minutes — using nothing but plain-English conversation. The agent reads your data, writes and runs Python, renders charts, and synthesizes findings — the full agent loop in action.

This module goes from a raw spreadsheet to a written recommendation with charts in under ten minutes — using nothing but plain-English conversation. We use a spreadsheet as an example, but other data sources can be used similarly. Module 5 discusses databases.

3.1 The scenario

You are a regional VP at a retail company. Q4 numbers just came in. The question on the table: which regions should get additional marketing budget, and which product lines should be expanded or cut?

The data is the Kaggle “Sample Superstore” dataset — roughly 10,000 orders across four U.S. regions, with columns for sales, profit, discount, product category, and customer segment. You can download it here: superstore.csv.

An analyst would take half a day to work through this. With a coding agent, you can do it in ten minutes.

3.2 Step 1: Open and orient

Place the CSV file in a folder and open your coding agent in that folder. Then ask:

Describe the data in this folder.

The agent reads the CSV into a dataframe and reports back: 10,000 rows, 21 columns, four regions, three product categories, a date range from 2014 to 2017, total sales of roughly $2.3 million with an average profit margin around 12%. It may also flag surprises — sub-categories that are losing money, or a disproportionate concentration of sales in one region.

This orienting step catches formatting issues (dates parsed as strings, currencies with dollar signs, encoding problems) and surfaces the structure of the data before you start asking pointed questions.

3.3 Step 2: Ask decision-relevant questions

Frame every question around the decision, not around the data. For the budget decision:

Which region has the highest profit margin, and which has the lowest?

Show quarterly sales trends by region. Are any accelerating or declining?

Which sub-categories are losing money? How much?

Each of these questions connects directly to the budget allocation decision. The first identifies where margin is strongest. The second reveals momentum — a region with declining sales may not deserve more investment regardless of its current margin. The third pinpoints money-losers that could be cut.

3.4 Step 3: Visualize the tradeoffs

Ask the agent to create charts that make the decision visual.

“Quarterly profit by region” calls for a line chart, because you are looking at trends over time. “Profit by sub-category, sorted” calls for a bar chart, because you are comparing categories. “Discount rate vs. profit margin” calls for a scatter plot, because you are looking for a relationship between two continuous variables.

Once you have a chart, iterate. Ask the agent to highlight money-losers in red, add a break-even line, or annotate the December spike. Each follow-up takes seconds. If an annotation overlaps a data point, you can ask the agent to view its own chart and fix the layout — it can see images and adjust the code accordingly.

3.5 Step 4: Build the recommendation

The final step is synthesis. Ask the agent to turn the analysis into a recommendation.

Which two regions should get increased marketing budget and why?

Which sub-categories should we discontinue? Quantify the savings.

Write a one-page executive memo with the recommendation and supporting charts.

In ten minutes you have gone from a raw CSV to a written recommendation backed by evidence. No code was written by you. No analyst was involved. No one waited three days for a dashboard.

3.6 What just happened

Behind the scenes, the agent read the CSV into Python, wrote and ran code for each question, rendered charts with Python, and synthesized findings into a memo. You never see the code unless you ask for it.

3.7 Working with multiple files

The Superstore walkthrough used a single CSV. Real decisions rarely come from a single file. A more common scenario: five regional managers each submit a Q1 sales spreadsheet, and you need a consolidated view.

Download aggregation.zip and extract it into a folder. You will find five Excel files — one per region — each with columns for date, product, units sold, revenue, and cost.

Open your coding agent in that folder and start the same way:

Describe the data in this folder.

The agent reads all five files, reports the column names and row counts for each, and flags any inconsistencies (a column named “Revenue” in one file and “Total Revenue” in another, or a date formatted as text). It standardizes the columns and merges everything into a single dataset without you writing a line of code.

Now ask cross-file questions:

Which region had the highest total revenue in Q1?

Compare profit margins across regions.

Show monthly revenue trends by region as a line chart.

Every follow-up works across the combined dataset. You can ask for comparisons, rankings, and charts exactly as you did with the single Superstore file. The only difference is the starting point: instead of one file, you pointed the agent at a folder of files.

This pattern scales. Whether it is five files or fifty, the workflow is the same: place the files in a folder, let the agent orient itself, then ask your questions.

3.8 Combining different file types

The next step up is working across file formats. Consider an invoice reconciliation scenario: your accounting team has an Excel spreadsheet of payments, a PDF of vendor invoices, and a Word document summarizing disputed charges.

Download invoice-reconciliation.zip and extract it into a folder. You will find three files:

  • payments.xlsx — a spreadsheet listing every payment made this quarter, with vendor name, amount, date, and reference number
  • invoices.pdf — a set of vendor invoices with invoice numbers, dates, amounts, and descriptions
  • disputes.docx — a memo from the finance team flagging specific invoices that are under dispute, with reasons

Open your coding agent in that folder and ask:

Read all files in this folder and summarize what each one contains.

The agent reads Excel, PDF, and Word natively. It will describe the structure of each file — rows and columns for the spreadsheet, extracted tables and text from the PDF, and the narrative content of the Word document.

Now ask it to reconcile:

Match each invoice to its corresponding payment. Flag any invoices that have no matching payment and any payments that don’t match an invoice.

This is where the real value appears. The agent handles the messy matching — vendor names that are slightly different between files, date formats that don’t align, reference numbers in one file that correspond to invoice numbers in another. It produces a reconciliation table showing matched records, unmatched invoices, and unmatched payments.

Then layer in the dispute information:

Which of the unmatched invoices are under dispute according to the memo? Produce a final reconciliation summary.

The agent cross-references the Word document’s dispute notes with the unmatched items and produces a clean summary: invoices paid, invoices unpaid but not disputed, invoices under active dispute, and payments with no matching invoice.

This pattern — extract, match, reconcile — is the foundation of due diligence reviews, compliance audits, and month-end close processes. The file formats vary, but the workflow is the same: point the agent at the files, ask it to read and reconcile, then iterate until the summary is complete.

3.9 Exercises

Exercise 1: The Superstore Walkthrough

This is the core exercise for Module 3. Download superstore.csv and place it in a folder. Open your coding agent in that folder and work through the four steps described above.

First, ask the agent to describe the data. Confirm the row count, column names, date range, and financial summary. Second, ask three decision-relevant questions tied to the marketing budget allocation decision. Third, create at least one line chart (for trends), one bar chart (for comparisons), and one scatter plot (for relationships). Iterate on at least one chart by adding annotations or improving the layout. Fourth, ask the agent to write a one-page executive memo with the recommendation and supporting charts.

Bring your dataset and memo to Module 4 — you will use them as thinking-partner exercises.

Exercise 2: Data Detective

Your colleague has made several claims about the Superstore data. Use your coding agent to verify or debunk each one.

Claim 1: Discounts over 30% always increase total profit. Claim 2: The East region has the best profit margins. Claim 3: Office Supplies is the most profitable category. Claim 4: Q4 is always the best quarter.

For each claim, ask the agent to test it, show the evidence, and write one sentence stating whether the claim is confirmed or debunked, with the supporting number.

Exercise 3: Employee Attrition Walkthrough

You are an HR director at a mid-size company. Employee turnover has been rising and the CEO wants to know why people are leaving and what to do about it.

Download employee-attrition.csv and place it in a folder. Open your coding agent and work through the same four steps from this module.

First, ask the agent to describe the data. How many employees are in the dataset? What is the overall attrition rate? What departments and job roles are represented?

Second, ask decision-relevant questions: Which departments have the highest attrition? Does overtime correlate with leaving? How does monthly income differ between employees who stay and those who leave? Are employees with longer commutes more likely to leave?

Third, create at least three charts: a bar chart comparing attrition rates by department, a box plot of monthly income for stayers vs. leavers, and a chart of your choice that reveals a pattern you find interesting.

Fourth, ask the agent to write a one-page memo recommending three retention interventions, backed by the data.

Exercise 4: Attrition Data Detective

Your VP of People Operations has made several claims about the employee attrition data. Use your coding agent to verify or debunk each one.

Claim 1: Employees who work overtime leave at more than double the rate of those who don’t. Claim 2: Job satisfaction has no meaningful relationship with attrition. Claim 3: Younger employees (under 30) are the most likely to leave. Claim 4: Employees who haven’t been promoted in the last two years leave at higher rates.

For each claim, ask the agent to test it, show the evidence, and write one sentence stating whether the claim is confirmed or debunked, with the supporting number.

Exercise 5: The Weekly Report

Identify a recurring report your team produces — weekly sales, monthly KPIs, quarterly review — and try to automate it.

Place the source data in a folder and open your coding agent. Ask it to produce the full report: charts, tables, and a narrative summary. Then compare: how long does this report normally take to produce? How long did AI take? What is missing from the AI version?

Exercise 6: Regional Sales Consolidation

Download aggregation.zip and extract it into a folder. You will find five Excel files, one for each region’s Q1 sales data.

Open your coding agent in the folder and work through these steps:

  1. Ask the agent to read all files and describe the data. Confirm the column names, row counts, and any inconsistencies it finds.
  2. Ask: which region had the highest total revenue? Which had the best profit margin? Are there products that sell well in one region but poorly in another?
  3. Create at least two charts: a bar chart comparing total revenue by region and a chart of your choice that reveals a cross-region pattern.
  4. Ask the agent to write a short recommendation: based on Q1 results, where should the company focus Q2 marketing spend, and why?
Exercise 7: Invoice Reconciliation

Download invoice-reconciliation.zip and extract it into a folder. You will find three files: a payments spreadsheet (Excel), a set of vendor invoices (PDF), and a dispute memo (Word).

Open your coding agent in the folder and work through these steps:

  1. Ask the agent to read all three files and summarize each one.
  2. Ask it to match invoices to payments and flag any that don’t match.
  3. Ask it to cross-reference unmatched invoices with the dispute memo.
  4. Ask for a final reconciliation summary: invoices paid, invoices unpaid but not disputed, invoices under dispute, and payments with no matching invoice.

When you are done, you should have a clean reconciliation table that an accountant could use to close out the quarter.
