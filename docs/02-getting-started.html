2  Getting Started

Module 1 explained what AI agents are and why they are transforming business intelligence. This module gets you set up to use one. By the end, you will have a coding agent running on your machine, ready to analyze data, write code, and produce documents from plain-English conversation.

2.1 Code execution environments

AI can write and run code in three different environments, each with different strengths.

A cloud sandbox is a temporary computing environment hosted by the AI provider. Claude Chat and ChatGPT both offer this: you upload a file, the chatbot processes it on a remote server, and you download the results. Cloud sandboxes are convenient for quick, ad-hoc analysis, but your data leaves your machine and the session is ephemeral.

A virtual machine is a sandboxed environment that runs on your own computer. Claude Cowork is an example: it spins up a VM locally, giving AI a full computing environment with file access and persistence across sessions. Virtual machines are good for longer batch work, and because they run on your machine, your data stays local.

A local environment runs directly on your machine. The Claude Desktop app is the primary example: the agent writes code that executes right where you work, with full access to your files, databases, and network. Nothing is uploaded. This is what we use throughout this course.

The key takeaway is that AI does not need a special platform to analyze your data. It runs where you already work — on your laptop, with your files. The rest of this module shows how to set that up.

2.2 Installing Python and Claude Desktop

The coding agent writes and runs Python behind the scenes, so both Python and the Claude Desktop app must be installed on your machine. We provide installers that handle everything — Python, Claude Desktop, and skills:

Windows — download the Windows installer. Right-click INSTALL.bat and select Run as administrator. The script installs Python 3.12 (auto-detecting your processor architecture), Claude Desktop, and the skills used in this course.

Mac — download the Mac installer. Open Terminal and run:

chmod +x ~/Downloads/install.sh
~/Downloads/install.sh

The script installs Homebrew (if needed), Python 3.12, Claude Desktop, and skills.

After either installer finishes, open Claude Desktop and sign in with your Anthropic account. Claude Code requires either a Pro subscription at $20 per month (which includes a generous usage allowance) or an Anthropic API key with credits.

Installing Python manually

If the installer does not work for Python, you can install it separately. Download Python 3.10 or later from python.org. On Windows, check “Add Python to PATH” during installation — this is in small print on the bottom of the very first installation screen. On Mac, you can also install via Homebrew with brew install python. After installing, verify by opening a terminal and running:

python --version

You should see something like Python 3.12.4. If the command is not found, restart your terminal or check that Python was added to your PATH.

Installing Claude Desktop manually

If the installer does not work for Claude Desktop, download it directly from claude.com/download and run the installer.

2.3 Using the Code tab

The Claude Desktop app has three tabs in the sidebar: Chat, Cowork, and Code. Click the Code tab. This is where you will do all of the work in this course.

The Code tab runs the Claude Code engine — a coding agent with full access to your local files, databases, and network. This is the agent architecture described in Module 1: an LLM connected to tools through an execution layer, running the plan-execute-observe-iterate loop autonomously. It reads files, writes and runs Python, renders charts, and produces documents, all from plain-English conversation.

To get started, click Code in the sidebar, then select a folder as your working directory. Choose a model (Sonnet is fast and capable; Opus is more thorough) and a permission mode. For the exercises in this course, “auto-accept” is the most convenient permission mode — it lets the agent run code without asking for approval on every step.

Try a quick test. Type:

Create a CSV file with five rows of sample sales data — columns for date, product, revenue, and profit. Then describe the data back to me.

The agent writes a Python script, creates the file in your working directory, reads it back, and reports the contents. You will see the file appear in your folder. That single exchange — natural language in, working file out — is the pattern for everything in this course.

Claude Code reads a file called CLAUDE.md in the project directory for persistent instructions. This is where you store context about your data, preferred output formats, and any domain-specific guidance the agent should follow every session.

Installing Python gives you the base language, but in practice most work relies on additional libraries — code packages for data analysis, charting, web apps, and more. At various points in this course, we will ask you to tell the coding agent to install specific libraries. If you skip that step, the agent will notice when it tries to run code that requires them and will ask you for permission to install what it needs. Just grant permission if that happens.

2.4 Alternative coding agents

The Code tab in Claude Desktop is what we use in this course. Under the hood it runs Claude Code — Anthropic’s coding agent engine. Claude Code is also available as a standalone terminal tool for users who prefer a command-line interface. Google and OpenAI offer similar terminal-based coding agents:

Gemini CLI is Google’s equivalent. It requires Node.js version 18 or higher and is installed with npm install -g @google/gemini-cli. It offers a free tier with 60 requests per minute and 1,000 requests per day. It reads a GEMINI.md file for project-level instructions, serving the same purpose as Claude Code’s CLAUDE.md.

OpenAI Codex is an open-source terminal agent that connects to OpenAI’s models. It is installed with npm install -g @openai/codex (requires Node.js 22 or higher). OpenAI also offers a native desktop app at openai.com/codex. Codex requires a ChatGPT Plus subscription or an OpenAI API key. It reads project instructions from an AGENTS.md file.

All three agents read local files, write Python, and return results in conversation. The workflows in this course apply to any of them. The key differences are cost (Claude Code requires a $20/month subscription or API credits; Gemini CLI’s free tier covers most individual use; Codex is open source but OpenAI API calls are billed per token), context windows (Gemini supports up to one million tokens; Claude Code ranges from 200,000 to over one million; Codex supports 192,000), and sandboxing (Codex uses an OS-level sandbox by default; Claude Code and Gemini CLI rely on permission prompts).

2.5 Skills

Out of the box, the coding agent can read files, write code, and run it. But you can extend what it knows how to do by installing skills — packaged instructions that teach the agent a specific workflow.

Claude Code has a skill system built in. A skill is a folder containing a markdown file that describes the task, the steps, and the output format. When you invoke a skill, Claude Code reads the instructions and follows them. The installer already set up four skills that are particularly useful for business work:

  • xlsx — read, create, and edit Excel spreadsheets with formulas, formatting, and charts.
  • docx — create and edit Word documents with tracked changes, comments, and formatting.
  • pptx — create and edit PowerPoint presentations.
  • skill-creator — a guide for building your own skills. You will use this in Module 5 when you create a database skill.

You can verify the skills are installed by opening the Code tab and typing / — the available skills appear as slash commands.

2.6 Setting up the Anthropic API

Later modules include demo apps that call Claude Sonnet via the Anthropic API. You will need an API key.

  1. Sign up at console.anthropic.com and add a payment method.

  2. Create an API key under Settings → API Keys.

  3. Set the key as an environment variable so the demo apps can find it:

    Mac / Linux — add to your shell profile (.bashrc or .zshrc) to persist:

    export ANTHROPIC_API_KEY="sk-ant-..."

    Windows — run once in a terminal, then restart the terminal:

    setx ANTHROPIC_API_KEY "sk-ant-..."

API calls are billed per token. A short request like the test below costs a fraction of a cent; a longer coding-agent session analyzing a dataset might cost a few cents to a few dollars depending on the length of the conversation. Current prices are listed at anthropic.com/pricing. To verify your key works, first install the Anthropic Python library by running pip install anthropic in a terminal or by asking your coding agent to install it. Then save this as test_anthropic.py and run it with python test_anthropic.py:

from anthropic import Anthropic

client = Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=[{"role": "user", "content": "Say hello in one sentence."}],
)

print(response.content[0].text)

If you see a greeting, the key is working and you are ready for the demo apps in Modules 5 and beyond.

2.7 Other interfaces

The Chat and Cowork tabs in Claude Desktop serve different purposes from the Code tab. Chat is a standard conversation interface for questions, writing, and image analysis — no code execution. Cowork runs tasks in a sandboxed virtual machine: it can organize files, draft reports, and process documents, but in a more isolated environment than the Code tab. Cowork is useful for batch knowledge work where you want an extra layer of safety.

Google Antigravity is an agent-first IDE — a fork of VS Code designed around AI collaboration. It is currently in free preview. Rather than adding AI to an existing editor, it was built from the ground up for agent-driven workflows, with the agent able to navigate files, run code, and iterate on changes directly in the editor.

VS Code with AI extensions is the most common setup for developers who already use VS Code. Claude Code, Gemini CLI, and Codex all have VS Code extensions that embed the agent directly in the editor.

Exercise 1: Install and Test

Run the installer for your platform and open Claude Desktop. Sign in, click the Code tab, select a folder, and confirm it is working by asking it to print “Hello, world” in Python.

Then ask it to create a simple CSV file with five rows of sample sales data (date, product, revenue, profit) and describe the data back to you. Verify that the file was created in your folder and that the description is accurate.

Exercise 2: Instruction Files

Create a CLAUDE.md file in a folder containing the sample CSV from Exercise 1. Add two instructions: a preferred chart style (for example, “use a dark background with white text for all charts”) and a default analysis behavior (for example, “always show the code you run”).

Open the Code tab in that folder and ask the agent to create a chart. Does it follow your instructions? Modify the instruction file and ask again.
